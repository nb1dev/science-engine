## 1. File Overview & Pipeline Flow

### 1.1 The Five Output Files

| # | File | Path Pattern | Generated By | Role |
|---|------|-------------|--------------|------|
| 1 | **Microbiome Analysis Master** | `reports/reports_json/microbiome_analysis_master_{id}.json` | `science-engine/report/generate_report.py` → `build_all()` | Core science output: scores, guilds, metabolic dials, root causes, action plan. **Ground truth for all microbiome data.** |
| 2 | **Formulation Master** | `reports/reports_json/formulation_master_{id}.json` | `science-engine/formulation/stages/s01–s09` | Complete supplement formulation: mix selection, prebiotics, vitamins, delivery formats. **Ground truth for all supplement data.** |
| 3 | **Questionnaire** | `questionnaire/questionnaire_{id}.json` | Platform API (user input) | Client-reported demographics, goals, symptoms, diet, medical history. **Ground truth for all client context.** |
| 4 | **Health Report Interpretations** | `reports/reports_json/health_report_interpretations_{id}.json` | `science-engine/report/generate_health_report.py` → `process_sample()` | **Front-end bridge file.** Pre-computed UI-ready data: circle scores, cards, timelines, supplement cards, goal cards, root cause analysis. |
| 5 | **Narrative Report** | `reports/reports_md/narrative_report_{id}.md` | `science-engine/report/generate_narrative_report.py` | LLM-generated deep clinical narrative (10 sections). Used for PDF report, not directly for front-end. |

### 1.2 Data Flow

```
Raw Bioinformatics Data
  (MetaPhlAn, GMWI2, guild CSV, metrics TXT)
        │
        ▼
┌─────────────────────────┐
│  Microbiome Analysis    │◄── Questionnaire JSON (for context)
│  Master JSON            │
│  (scoring, guilds,      │
│   metabolic dials,      │
│   root causes, etc.)    │
└────────┬────────────────┘
         │
         ├──────────────────────┐
         ▼                      ▼
┌─────────────────┐   ┌──────────────────────┐
│  Formulation    │   │  Health Report        │
│  Master JSON    │   │  Interpretations JSON │
│  (supplements)  │   │  (front-end bridge)   │
└────────┬────────┘   └──────────┬───────────┘
         │                       │
         ▼                       ▼
┌─────────────────┐   ┌──────────────────────┐
│  Narrative      │   │  HTML Health Report   │
│  Report MD/PDF  │   │  (front-end)          │
└─────────────────┘   └──────────────────────┘
```

### 1.3 Sample ID Convention

All files use the **kit_code** as the sample identifier (e.g., `1421029282376`). This is a numeric string that uniquely identifies a physical sample kit. It appears in filenames and as `sample_id` or `kit_code` inside the JSONs.

## 2. Microbiome Analysis Master JSON

**File:** `microbiome_analysis_master_{id}.json`
**Generator:** `science-engine/report/generate_report.py` → `build_all()`
**Depends on:** Raw bioinformatics metrics, guild CSV, questionnaire JSON

This is the **primary science output** — the single source of truth for all microbiome analysis data. Every other report file derives from or references data in this file.

---

### 2.1 `report_metadata`

Top-level metadata about the report generation.

| Key | Type | Description | Example | Narrative Use |
|-----|------|-------------|---------|---------------|
| `sample_id` | string | Kit code / sample identifier | `"1421029282376"` | Linking reports together |
| `report_date` | string (ISO date) | Date the report was generated | `"2026-03-10"` | Display date on report |
| `algorithm_version` | string | Version of the scoring algorithm | `"3.0"` | Versioning / reproducibility |
| `generated_at` | string (ISO datetime) | Exact generation timestamp | `"2026-03-10T20:34:34.301870"` | Audit trail |
| `llm_model` | string | Bedrock model used for LLM-generated narratives | `"eu.anthropic.claude-sonnet-4-20250514-v1:0"` | Provenance tracking |

| Key | Type | Description | Narrative Use |
|-----|------|-------------|---------------|

| `key_finding.scientific` | string | The single most important finding (scientific register) | Key insight card |

| `recovery_potential.non_expert` | string | Plain-language recovery potential | **"Can this be fixed?" section** |

### 2.3 `overall_score`

The composite microbiome health score (0–100) broken into 5 pillars.

#### Top-level score

| Key | Type | Range | Description | Narrative Use |
|-----|------|-------|-------------|---------------|
| `total` | float | 0–100 | Composite microbiome health score | **Main score display** |
| `band` | string | `"Critical"` / `"Poor"` / `"Fair"` / `"Good"` / `"Excellent"` | Score classification band | Score badge / label |
| `description` | string | — | Human-readable description of the band | Subtitle under score |

#### `pillars` — The 5 scoring dimensions

Each pillar has the same structure:

| Key | Type | Description |
|-----|------|-------------|
| `pillars.{name}.score` | float | Points earned for this pillar |
| `pillars.{name}.max` | float | Maximum possible points for this pillar |
| `pillars.{name}.scientific` | string | Scientific description of what this pillar measures |
| `pillars.{name}.non_expert` | string | Plain-language description |

**The 5 pillars:**

| Pillar Key | Max Points | What It Measures |
|------------|-----------|-----------------|
| `health_association` | 20 | How health-associated the species composition is (GMWI2 model of 155 taxa) |
| `diversity_resilience` | 20 | Shannon diversity + guild-level evenness = species richness and structural redundancy |
| `metabolic_function` | 20 | Composite of 4 CLR diagnostic ratios (CUR, FCR, MDR, PPR) |
| `guild_balance` | 30 | 6 functional guilds evaluated by abundance + CLR, weighted by ecological dependency |
| `safety_profile` | 10 | Absence of dysbiosis markers, extreme overgrowth, BCFA pathways |

#### `score_drivers`

| Key | Type | Description | Narrative Use |
|-----|------|-------------|---------------|
| `score_drivers.strongest.pillar` | string | Key of the highest-performing pillar | "Your strongest area is..." |
| `score_drivers.strongest.label` | string | Human-readable name | Display label |
| `score_drivers.strongest.pct_of_max` | int | Percentage of max achieved | Progress bar |
| `score_drivers.weakest.pillar` | string | Key of the lowest-performing pillar | "Your biggest opportunity is..." |
| `score_drivers.weakest.label` | string | Human-readable name | Display label |
| `score_drivers.weakest.pct_of_max` | int | Percentage of max achieved | Progress bar |
| `score_drivers.key_note` | string | Pre-written summary of strongest area | Quick insight text |

#### `details` — Raw scoring breakdown (P1–P5)

These are the detailed calculation inputs for each pillar. Primarily for debugging/validation, but can be used for advanced visualizations.

**P1 (Health Association):**

| Key | Type | Description |
|-----|------|-------------|
| `details.P1.hf_score` | float | Health Fraction contribution |
| `details.P1.concordance` | float | Agreement between presence and abundance signals |
| `details.P1.sb_score` | float | Signature Balance score |
| `details.P1.br_score` | float | Bias Ratio score |
| `details.P1.z_pts` | float | Z-score bonus points |

**P2 (Diversity & Resilience):**

| Key | Type | Description |
|-----|------|-------------|
| `details.P2.shannon_score` | float | Points from Shannon diversity |
| `details.P2.avg_guild_J` | float | Average Pielou's J across all guilds (0–1) |
| `details.P2.guild_resilience` | float | Composite guild resilience score |
| `details.P2.guilds_counted` | int | Number of guilds included in calculation |

**P3 (Metabolic Function) — 4 CLR ratios:**

Each ratio under `details.P3.{RATIO}`:

| Key | Type | Description |
|-----|------|-------------|
| `.raw` | float | Raw calculated ratio value |
| `.resolved` | float | Value after NaN/edge-case resolution |
| `.score` | float | Points awarded (0–5 each, 20 max total) |

Ratios: `CUR`, `FCR`, `MDR`, `PPR` (see §2.6 for full definitions).

**P4 (Guild Balance):**

| Key | Type | Description |
|-----|------|-------------|
| `details.P4.abundance_scores.{guild}` | float | Points from abundance position in range |
| `details.P4.clr_modifiers.{guild}` | float | CLR-based modifier (bonus/penalty) |
| `details.P4.total_before_cap` | float | Sum before capping to max 30 |

Guild keys: `cross`, `fiber`, `butyrate`, `bifido`, `proteo`, `mucin`

**P5 (Safety Profile):**

| Key | Type | Description |
|-----|------|-------------|
| `details.P5.markers.F_nucleatum` | float | Score for F. nucleatum absence (0–2) |
| `details.P5.markers.S_gallolyticus` | float | Score for S. gallolyticus absence (0–2) |
| `details.P5.markers.P_anaerobius` | float | Score for P. anaerobius absence (0–2) |
| `details.P5.markers.E_Shigella` | float | Score for E-S absence (0–2) |
| `details.P5.markers.total` | float | Sum of marker scores (max 8) |
| `details.P5.extreme_penalty` | float | Penalty for extreme guild overgrowth |
| `details.P5.bcfa_penalty` | float | Penalty for BCFA pathway detection |
| `details.P5.smithii_penalty` | float | Penalty for M. smithii presence |
| `details.P5.safety_base` | float | Base safety score before markers |

### 2.5 `safety_profile`

Detection status of known dysbiosis-associated organisms and pathways.

#### `dysbiosis_markers`

Each marker has:

| Key | Type | Description |
|-----|------|-------------|
| `.abundance` | float | Relative abundance percentage (0.0 if not detected) |
| `.status` | string | `"Not detected"` / `"Detected at X%"` |

**The 4 markers:**

| Marker Key | Organism | Clinical Relevance |
|------------|----------|-------------------|
| `F_nucleatum` | *Fusobacterium nucleatum* | Inflammatory/periodontal pathogen, colorectal cancer association |
| `S_gallolyticus` | *Streptococcus gallolyticus* | Colorectal cancer-associated organism |
| `P_anaerobius` | *Peptostreptococcus anaerobius* | Colorectal neoplasia marker |
| `E_Shigella` | *Escherichia-Shigella* | Enteropathogenic potential |

#### Other safety fields

| Key | Type | Description |
|-----|------|-------------|
| `M_smithii_abundance` | float | *Methanobrevibacter smithii* abundance (methane producer) |
| `bcfa_pathways_detected` | int | Count of branched-chain fatty acid pathways detected |

### 2.6 `metabolic_function`

The 4 metabolic "dials" that characterize the ecosystem's operating state.

#### `dials` — Each dial has:

| Key | Type | Description |
|-----|------|-------------|
| `.state` | string | State classification (see table below) |
| `.label` | string | Human-readable label for the state |
| `.value` | float | The resolved ratio value |
| `.raw_value` | float | The raw calculated ratio value |
| `.metric` | string | Ratio name (`CUR`, `FCR`, `MDR`, `PPR`) |
| `.description` | string | **Plain-language explanation of what this means** |

**The 4 Dials and Their States:**

| Dial Key | Ratio | Formula | Favorable | States |
|----------|-------|---------|-----------|--------|
| `main_fuel` | CUR (Carbohydrate Utilization Ratio) | `avg(Fiber_CLR, Bifido_CLR) − Proteo_CLR` | Positive (>+0.5) | `carb_driven` / `mixed` / `protein_driven` |
| `fermentation_efficiency` | FCR (Fermentation Completion Ratio) | `avg(Butyrate_CLR, Cross_CLR) − Bifido_CLR` | Positive (>+0.3) | `efficient` / `moderate` / `bottleneck` |
| `mucus_dependency` | MDR (Mucus Dependency Ratio) | `Mucin_CLR − Fiber_CLR` | Negative (<−0.5) | `diet_fed` / `mixed_feeding` / `mucus_dependent` |
| `putrefaction_pressure` | PPR (Putrefaction Pressure Ratio) | `Proteo_CLR − Butyrate_CLR` | Negative (<−0.5) | `scfa_dominant` / `mixed_output` / `putrefactive` |

#### `interpretation`

| Key | Type | Description |
|-----|------|-------------|
| `interpretation.scientific` | string | Full scientific interpretation of all 4 dials together |
| `interpretation.non_expert` | string | **Plain-language metabolic summary — excellent for dashboard hero text** |

### 2.8 `bacterial_groups`

The 6 functional guilds — detailed status for each.

**Guild keys and their display names:**

| Key in JSON | Display Name | Guild Type | Role |
|-------------|-------------|------------|------|
| `Cross-Feeders` | Intermediate Processors | beneficial | Convert primary fermentation intermediates into substrates for butyrate producers |
| `Fiber Degraders` | Fiber-Processing Bacteria | beneficial | Primary breakdown of complex dietary polysaccharides |
| `Butyrate Producers` | Gut-Lining Energy Producers | beneficial | Terminal SCFA production — butyrate for colonocytes |
| `Bifidobacteria` | Bifidobacteria | beneficial | Bifidus shunt fermentation, acetate/lactate production, immune modulation |
| `Proteolytic Guild` | Protein-Fermenting Bacteria | contextual | Protein fermentation — potentially harmful at high levels |
| `Mucin Degraders` | Mucus-Layer Bacteria | contextual | Mucin glycoprotein degradation — maintenance function |

**Per-guild fields:**

| Key | Type | Description | Narrative Use |
|-----|------|-------------|---------------|
| `.abundance` | float | Relative abundance percentage | **"Your X bacteria are at Y%"** |
| `.healthy_range` | string | Reference range (e.g., `"30-50%"`) | **"...compared to the healthy range of Z"** |
| `.status` | string | `"Below range"` / `"Within range"` / `"Above range"` | Status badge |
| `.priority_level` | string | `"1A"` / `"1B"` / `"2A"` / `"Monitor"` etc. | Intervention priority |
| `.clr` | float | Centered Log-Ratio competitive position | Advanced detail |
| `.clr_status` | string | `"Balanced"` / `"Slightly depleted"` / `"Depleted"` / `"Enriched"` etc. | Competition label |
| `.evenness` | float | Pielou's J within this guild (0–1) | Redundancy measure |
| `.evenness_status` | string | `"Low redundancy"` / `"Moderate redundancy"` / `"High redundancy — resilient"` | Resilience label |
| `.scientific_interpretation` | string | Full scientific interpretation | Expert-facing text |
| `.client_interpretation` | string | **Plain-language interpretation — primary text for front-end guild cards** | **Client-facing guild card** |

### 2.11 `root_causes`

Deep causal analysis of why the microbiome is in its current state.

#### `diagnostic_flags[]`

Array of flagged conditions:

| Key | Type | Description |
|-----|------|-------------|
| `.flag` | string | Human-readable flag description |
| `.guild` | string | Client-facing guild name |
| `.severity` | string | `"moderate"` / `"significant"` / `"critical"` |
| `.direction` | string | `"below"` / `"above"` |
| `.metric_detail.actual` | float | Actual abundance |
| `.metric_detail.range_min` | float | Range minimum |
| `.metric_detail.range_max` | float | Range maximum |
| `.metric_detail.fraction_of_min` | float | Actual as fraction of minimum (e.g., 0.51 = 51% of minimum) |

#### `trophic_impact`

Cascade effects through the metabolic chain:

| Key | Type | Description |
|-----|------|-------------|
| `.cascade_impacts[]` | array | List of downstream impacts |
| `.cascade_impacts[].type` | string | `"substrate_limitation"` / `"competitive_displacement"` / `"metabolic_bottleneck"` |
| `.cascade_impacts[].title` | string | Human-readable title |
| `.cascade_impacts[].description` | string | **Plain-language description of cascade effect** |
| `.cascade_impacts[].affected_guilds[]` | array of strings | Client-facing guild names affected |
| `.primary_bottleneck` | string | Primary bottleneck type |
| `.total_impacts` | int | Number of cascade impacts |

#### `reversibility`

|-----|------|-------------|---------------|

| `.description` | string | **Plain-language recovery description** | **Recovery section text** |
| `.estimated_timeline` | string | Expected recovery time (e.g., `"12-16 weeks"`) | **Timeline display** |
| `.transition_caution` | boolean | Whether special caution is needed during transition | Warning flag |

## 8. AI Tool Usage Guide

### 8.1 General Principles

1. **Start with the Interpretations JSON** — it contains pre-computed, UI-ready data. Use it as the primary source for front-end content.
2. **Fall back to Analysis Master for depth** — when you need scientific detail, dual-register text (scientific + non_expert), or raw metrics.
3. **Use Formulation Master for supplement explanations** — the `component_registry[]` is the best source for "why is this in my formula?" content.
4. **Use Questionnaire for personalization** — goals, stress level, sleep quality, diet, medical history.
5. **Use Narrative Report for deep scientific content** — when generating educational or expert-facing content.

### 8.3 Dual-Register Text Pattern

Many fields in the Analysis Master JSON provide both scientific and non-expert text:

```json
{
  "scientific": "Technical description with metrics and terminology",
  "non_expert": "Plain-language description suitable for clients"
}
```

**Rule:** Always use `.non_expert` for client-facing front-end content. Use `.scientific` only for expert-facing views or hover-over detail panels.

### 8.4 Data Freshness Hierarchy

When the same data appears in multiple files, use this priority:

1. **Interpretations JSON** — most processed, front-end ready
2. **Analysis Master JSON** — ground truth for microbiome data
3. **Formulation Master JSON** — ground truth for supplement data
4. **Questionnaire JSON** — ground truth for client-reported data
5. **Narrative Report MD** — LLM-generated, may have minor discrepancies from regeneration
